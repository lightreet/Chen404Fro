# Chen404 前端功能审查与优化清单

更新时间：2026-05-28

## 审查范围

本次审查覆盖当前前端已有功能、最近新增功能、文档口径和前后端接口对齐情况。重点检查了路由、手写 API、Pinia store、主要页面、后台管理、PageHero 视觉系统、Sakura Radio、旅行地图、AI 聊天与配置、文件管理。

## 已落地能力

- 公开内容：`/` 首页、文章详情、归档、分类、标签、留言板、关于页、公开用户主页。
- 内容创作：独立文章编辑页，支持 Markdown、图片插入、封面上传、草稿/发布、AI 摘要与标签。
- 权限与个人中心：登录、注册、刷新 token、个人资料、受信申请、知友访问控制。
- 旅行纪念地图：管理员/知友访问，支持地点、图片、游记条目、坐标、访问时间范围、城市/省级 GeoJSON 地图数据与基础地图降级。
- Sakura Radio：公开音乐馆、默认电台、公开歌单、歌词展示、播放器状态共享、Live2D 小播放器、管理员曲目维护、歌单维护、AI 曲目信息补全。
- 后台管理：分类、站点配置、AI 助手、表情包、文件、受信申请。
- 文件管理：文件列表、引用状态、统计卡片、类型图表、详情抽屉、音乐音频/封面类型筛选。
- AI 与 Live2D：同步聊天、SSE 流式聊天、会话恢复、站内引用、相关推荐、快捷建议、小气泡和聊天面板回答分层。
- 视觉系统：统一 PageHero、HeroWave、功能页封面渐变过渡、樱花氛围层和独立工作台页面形态。

## 最近新增或调整

- 新增 `/music`、`/music/tracks/new`、`/music/tracks/:id/edit` 路由。
- 新增 `src/api/music.ts` 与 `src/stores/music-player.ts`。
- 新增 Live2D 音乐按钮和 `Live2DMusicPanel`。
- 音乐曲目编辑页改为不套全站顶部菜单的独立工作台。
- PageHero 底部过渡改为固定前景波浪 + 背层轻动画 + 内容区渐变，降低 SVG transform 露缝风险。
- 后台文件管理补齐音乐音频和音乐封面的前端筛选标签。

## 当前不足或未实现

- AI 工具里的 `webSearchEnabled` 仍是预留开关，前端没有真实联网搜索交互。
- OpenAPI SDK 已生成，但业务页面仍主要使用手写 API；类型和接口口径存在长期分叉风险。
- Sakura Radio 尚未提供歌单删除、播放统计、收藏/点赞、评论、播放历史、Media Session 系统媒体控制。
- 播放器只持久化音量和播放模式，不跨会话恢复当前曲目与播放进度。
- 音频加载失败时已有自动切歌逻辑，但全队列资源都不可用时仍需要更明确的停止和提示策略。
- 文件管理没有批量清理、批量转永久、批量删除、手动触发引用重建的完整前端操作流。
- 站点配置中仍有部分“后台可配、前台逐步接入”的运行时配置，需要按页面持续验收。
- 前端自动化测试覆盖偏薄，播放器、权限守卫、SSE、旅行地图、文件筛选等交互主要依赖构建和人工验证。

## 优化建议

- 先把音乐模块补齐最影响管理闭环的能力：歌单删除、歌单封面上传、音频失败终止策略、播放状态恢复。
- 为 PageHero 建立截图回归清单，覆盖首页、分类、归档、音乐馆、受信申请、留言板、关于页和移动端。
- 将 `src/api/*.ts` 与 `src/types/index.ts` 中已稳定的模块逐步迁移到 OpenAPI 生成类型，先从音乐和文件管理这种新增模块开始。
- 文件管理前端补一个“维护任务”区，承接引用重建、未引用文件排查、临时文件清理策略说明和后端任务状态。
- Sakura Radio 可以后续接入 Media Session API，让浏览器/系统媒体键能控制播放。
- 对 `music-player`、路由守卫、AI SSE 消费和旅行地图数据加载补单元测试或 Playwright 冒烟测试。
- 对城市 GeoJSON 继续监控体积和加载耗时，必要时按省份懒加载或拆包。
