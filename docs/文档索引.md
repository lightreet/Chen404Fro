## 维护规则

### 适合写进 `README.md`

- 本地运行方式
- 环境变量
- 当前页面与权限
- 对接状态与已知边界

### 适合写进 `architecture.md`

- 目录结构
- 路由与守卫
- API 封装策略
- store/composable/module 职责
- 关键业务流

### 不再重复维护的内容

- 同一功能的“需求稿 + 实现稿 + 进度稿”三份并存
- 与代码已经脱节的旧规划
- 能直接从代码读取的低价值罗列

当功能发生变化时，优先更新 `README.md` 或 `architecture.md`，其他文档只保留补充说明或历史记录。
