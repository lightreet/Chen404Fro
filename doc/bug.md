# Chen404Fro Bug 清单

> 说明：本文件记录前端侧已确认的问题与处理状态，便于后续联调和回归。

## 待处理

### 1. 管理员路由守卫依赖本地缓存角色信息

- 状态：待处理
- 严重级别：中
- 影响范围：`src/router/index.ts`
- 问题描述：`requiresAdmin` 判断基于 `localStorage.user` 中的角色信息，理论上可通过篡改本地缓存进入后台页面路由。
- 影响：后端接口仍会拦截真实越权请求，因此不是服务端安全漏洞，但会造成前端 UI 层“假进入后台”的体验问题。
- 建议修复：进入管理员路由前增加一次基于真实登录态的校验，或由 Pinia / `/auth/info` 返回结果统一驱动权限状态。

### 2. 受信用户 / 好友管理仍缺少前端管理入口

- 状态：待处理
- 严重级别：中
- 影响范围：后台管理端
- 问题描述：后端已经具备 `PUT /admin/users/{id}/trust-level` 接口，但前端还没有对应的管理页面、表格操作入口或交互按钮。
- 影响：目前仍需手动调接口，无法在后台界面中直接维护受信用户。
- 建议修复：增加管理员用户列表页或在现有后台用户管理模块中补上“设为受信用户 / 取消受信用户”的操作。

## 已修复

### 1. 文章详情页无法正确区分 403 场景

- 状态：已修复
- 严重级别：中
- 原问题：访问受限文章时，前端容易显示为通用“文章不存在/加载失败”，无法明确区分“无权限访问”。
- 修复结果：文章详情页已根据后端真实 HTTP 状态码对 403 场景展示“当前文章仅对特定用户开放”等更准确提示。

### 2. 前端管理员判断逻辑不统一

- 状态：已修复
- 严重级别：中
- 原问题：不同位置使用不同角色判断方式，存在基于旧角色值的分散逻辑。
- 修复结果：已统一收敛到 `src/utils/permission.ts` 中的辅助函数，如 `isAdminUser()`、`isFriendUser()`、`getTrustLevelLabel()`。

### 3. 用户资料页未体现新的受信用户分层

- 状态：已修复
- 严重级别：低
- 原问题：个人中心无法展示“普通用户 / 好友 / 管理员”的当前身份层级。
- 修复结果：`Profile` 页面已增加角色和信任级别展示。

### 4. 文章编辑页无法设置可见性和评论策略

- 状态：已修复
- 严重级别：低
- 原问题：前端编辑页无法操作后端新增的 `visibility`、`commentPolicy` 字段。
- 修复结果：编辑页已补充对应高级设置表单项。

### 5. 文章详情页缺少基于后端权限的编辑入口控制

- 状态：已修复
- 严重级别：低
- 原问题：前端无法直接根据服务端权限结果决定是否显示“编辑文章”入口。
- 修复结果：文章详情页已根据后端返回的 `canEdit` 控制按钮展示。
